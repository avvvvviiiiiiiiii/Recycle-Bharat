# üìÑ PROJECT REQUIREMENT DOCUMENT (PRD)

## Project Name

**Digital Product Passport for Incentive-Driven E-Waste Management**

## Problem Statement

India faces a rapidly growing e-waste crisis due to short device lifecycles, lack of device-level tracking, dominance of informal recycling, low citizen participation, and fragmented government visibility. Existing systems track e-waste at an aggregated or regulatory level but fail to provide **end-to-end lifecycle transparency**, **economic incentives**, and **real-time governance data**.

---

## 1. Product Vision

To build a **digital, incentive-driven platform** that assigns a unique identity to every electronic device and tracks it from **ownership ‚Üí recycling request ‚Üí collection ‚Üí authorized recycling**, while rewarding citizens and collectors and enabling real-time government monitoring.

---

## 2. Objectives

* Enable **device-level lifecycle tracking**
* Incentivize **responsible disposal** by citizens
* Digitally integrate **kabadi/collectors** into formal systems
* Improve **authorized recycling compliance**
* Provide **real-time analytics** to government authorities
* Support **circular economy & SDG-12**

---

## 3. System Overview (High Level)

The system consists of:

* A **central backend platform**
* **Four role-based dashboards**
* A **state-based lifecycle engine**
* A **government analytics layer**

### Device Lifecycle States

```
ACTIVE
‚Üí RECYCLING_REQUESTED
‚Üí COLLECTOR_ASSIGNED
‚Üí COLLECTED
‚Üí DELIVERED_TO_RECYCLER
‚Üí RECYCLED (FINAL)
```

---

## 4. User Roles & Access Control (RBAC)

| Role                   | Access Level                                 |
| ---------------------- | -------------------------------------------- |
| Citizen                | Own devices, request recycling, track status |
| Kabadi / Collector     | Accept pickup, collect & deliver devices     |
| Recycler (Authorized)  | Manage requests, assign collectors, recycle  |
| Government / Authority | View analytics, compliance & trends          |
| System Admin           | Manage users, roles, configurations          |

# 5. FEATURE REQUIREMENTS (Dashboard-wise)

---

## A. Citizen Dashboard

### Core Purpose

Enable citizens to **register devices**, **request recycling**, **track progress**, and **earn rewards**.

### Features

**Authentication**

* Citizen registration & login
* Profile creation (minimal PII)

**Device Management**

* View all owned devices
* Device details:

  * Device ID
  * Device type
  * Status
  * Purchase/activation date

**Recycling Flow**

* Request recycling for a device
* View nearest authorized recycler
* Track real-time status updates

**Incentives & Recognition**

* Credit points for successful recycling
* Recycling history
* Social responsibility badge / acknowledgment

**Tracking & Transparency**

* Timeline view of recycling stages
* Notifications (request accepted, collected, recycled)

---

## B. Kabadi / Collector Dashboard

### Core Purpose

* Role-restricted access

**Pickup Management**

* View assigned pickup requests
* Pickup details:

  * Citizen ID
  * Device ID
  * Location (basic)

**Collection Flow**

* Enter/scan device ID
* Confirm pickup
* Update status to ‚ÄúCollected‚Äù

**Delivery Flow**

* Deliver device to assigned recycler
* Mark device as ‚ÄúDelivered‚Äù
